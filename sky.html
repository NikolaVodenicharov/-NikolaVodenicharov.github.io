Sky

Sky is a weather app with clean display, easy to use and straight to the point UI. Fast and responsive, without an overwhelming amount of data.

Description

The main display that is loaded after starting the app. With plain and simple search bar. While searching for a name of city a list with over a 100 000 cities to choose from appear. The result of search apper after selecting a city and clicking the search button. A tip... there is no need to search the city that we are currently in. We just have to slide to the right. The app will get the current location weather for us.

What is used in the project

MVVM facilitate separation of concern. Maintaining a clear detaching UI and logic. Making application easier to test, maintain and evolve. A domain driven design that is positioning the business logic and application models at the center. They do not depend of any data access of infrastructure. The dependency is inverted, implementation details depend of the core. This functionallity is achieved by abstraction and interfaces. Using the Volley library we are requesting a data for the weather from remote source. Using the provided api with the specific documentation from that source we can utilize it for our purposes. Using a Room database to store the weather results for reuse in the next few min. That is reducing the internet traffic from making request for weather forecast for the same city after we already have get one recently. The different sources of data are merged together at one source of truth. Repository responsibility to gather, keep up to date and give information to the rest of the app. The flow of data goes only through one place that manage it. Written with java for android the app have the bones of one of the best languages out there.
